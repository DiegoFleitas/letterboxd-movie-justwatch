## HIGH [4]

### 1. Rate limiter ineffective on Fly.io

`trustProxy: true` is missing from the Fastify constructor in `src/server/createServer.ts`. All requests arrive via Fly's reverse proxy, so `request.ip` resolves to the proxy address — every client shares one rate-limit bucket. The 30 req/min cap is non-functional.

**Fix:** Add `trustProxy: true` to Fastify constructor. Configure the rate limiter's `keyGenerator` to use `fly-client-ip`:

```typescript
keyGenerator: (request) => {
  return (request.headers["fly-client-ip"] as string) ?? request.ip;
},
```

### 2. `/api/sentry-test` unguarded

Inlining `registerFastifySentryTestRoute` into `registerFastifyAppApi` placed the route at the `app` scope, outside both the CSRF guard and the rate limiter (both scoped to the `apiRoutes` sub-plugin, lines 51–76 of `registerFastifyAppApi.ts`). Any unauthenticated GET request can exhaust Sentry quota or inject noise into the error dashboard.

**Fix:** Either gate on `NODE_ENV !== "production"` or require a secret token parameter. At minimum move inside the `apiRoutes` scope to inherit rate limiting.

### 3. Jackett HTTP parameter pollution

`src/server/controllers/alternativeSearch.ts` — title and year are concatenated into the Jackett URL with only spaces replaced by `+`. Characters like `&`, `=`, `?` pass through, allowing query parameter injection (e.g. overriding `Category`).

**Fix:** Use `encodeURIComponent` on the search query before appending to the URL.

### 4. No `max()` constraints on Zod schemas

`src/server/lib/apiSchemas.ts` — all text fields (`title`, `year`, `username`, `listUrl`, `country`, `listType`) are unbounded. Oversized inputs are passed to external APIs, logged, and used as cache key inputs, making the 256 MB VM trivially DoS-able.

**Fix:** Add `.max()` to every string field. Suggested limits: `title` 200, `year` 4, `username` 50, `listUrl` 300, `country` 10, `listType` 50.

## MEDIUM [4]

### 5. PostHog proxy unrate-limited

`/api/reversa/*` is registered directly on `app`, outside the `apiRoutes` sub-plugin that carries the rate limiter. Unlimited relaying of requests to PostHog.

**Fix:** Wrap in a scoped sub-plugin with its own rate limiter (suggested: 60 req/min).

### 6. Spoofable `x-forwarded-for` in PostHog proxy

`src/server/controllers/posthogProxy.ts` — the IP fallback chain trusts the client-supplied `x-forwarded-for` header. If `fly-client-ip` is absent, an attacker can forge the forwarded IP, poisoning PostHog geolocation analytics.

**Fix:** After fixing trustProxy (HIGH #1), rely only on `fly-client-ip` or `req.ip`; drop the `x-forwarded-for` fallback from incoming headers.

### 7. `unsafe-inline` in CSP script-src

`src/server/registerFastifyWiring.ts` line 30 — `'unsafe-inline'` negates XSS protection from CSP. Present because `injectRuntimeConfig` writes an inline `<script>` block.

**Fix:** Generate a per-request nonce, inject it into both the CSP header and the inline script tag. Interim: document the known gap.

### 8. `secure: true` hardcoded on session cookie

`src/server/registerFastifySessionPlugins.ts` — `secure: true` is unconditional, causing session cookies to be silently dropped in local HTTP dev.

**Fix:** `secure: process.env.NODE_ENV === "production"`.

## LOW [5]

### 9. Redis hostname leaked in 403 response

`src/server/lib/devApiGuard.ts` line 58 — the Redis hostname is included in the error body returned to clients. Log it server-side instead.

### 10. CSRF guard is a static header check

`x-requested-by: MovieJustWatch` is visible in the JS bundle. Safe given `origin: false` CORS, but not a true CSRF token. Document the dependency on CORS for this protection to hold.

### 11. `cross-spawn` ReDoS in dev dependencies

`cross-spawn < 6.0.6` (GHSA-3xgq-45jj-v275) present transitively via ESLint. Run `pnpm update` to pull in the patched version. The `electron` vulnerabilities via `react-devtools` are dev-only.

### 12. `listType` field not validated against an enum

`src/server/lib/apiSchemas.ts` — `listType` is `z.string().optional()` with no allowed-values check. It is incorporated into Redis cache key prefixes and logged. Constrain to an enum or remove if unused.

### 13. Axios log sanitization regex coverage

`src/server/lib/axios.ts` lines 31–32 — the redaction regex covers common key patterns. No action required, but verify any new external HTTP clients added in future use the shared axios helper or equivalent redaction.

## Security strengths confirmed

- SSRF controls on the generic proxy (`/api/proxy`): strict hostname allowlist, HTTPS enforcement, `new URL()` normalisation before comparison.
- Letterboxd URL validation constrained to `letterboxd.com` via tight regex before any outbound fetch.
- PostHog path allowlist blocks traversal (`..`, `%2e`, `%2E`) and enforces a prefix allowlist.
- CORS locked down (`origin: false`).
- Helmet configured with HSTS (1 year, includeSubDomains, preload), COOP, `frame-ancestors: none`.
- Axios and Letterboxd HTTP client both redact credential query params from logs.
- Dev routes production-gated via `isNodeProductionEnvironment()`, returning 404.
- PostHog proxy only forwards `content-type` from upstream — no response header injection.
- Session cookie flags: `httpOnly: true`, `sameSite: "strict"`.
